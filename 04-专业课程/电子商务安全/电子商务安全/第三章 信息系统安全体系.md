# 第三章 信息系统安全体系
根据《第三章 信息系统安全体系.pdf》内容，对问题的回答如下（**简洁、紧扣PPT原文**）：

* * *

### 1\. **安全体系结构提供的主要内容有哪些？**

*   提供**安全服务**（安全功能）与相应的**安全机制**在体系结构下的一般描述；
*   确定在体系结构内部**可以提供这些服务的位置**；
*   保证安全服务能**完全、准确地配置**，并在信息系统**全生命周期中持续维持**。

* * *

### 2\. **ISO开放系统互连安全体系结构中的五类安全服务、八类安全机制的内容及相互联系**

#### **五大类安全服务**：

1.  **鉴别服务**：包括对等实体鉴别和数据源鉴别；
2.  **访问控制服务**：限制对资源的非授权使用；
3.  **数据机密性**：包括连接、无连接、选择字段和通信业务流的机密性；
4.  **数据完整性**：包括带/不带恢复的连接完整性、选择字段完整性、无连接完整性等；
5.  **抗抵赖服务**：包括原发证明抗抵赖和交付证明抗抵赖。

#### **八类安全机制**：

1.  加密
2.  数字签名
3.  访问控制
4.  数据完整性
5.  鉴别交换（如口令、生物特征）
6.  通信业务流填充（对抗流量分析）
7.  路由控制（动态或预定路由）
8.  公证（由可信第三方验证数据性质）

#### **相互联系**：

*   安全机制是**实现安全服务的技术手段**。
*   例如：
    *   **加密** 可支持所有机密性服务和部分完整性服务；
    *   **数字签名** 支持数据源鉴别、完整性、抗抵赖；
    *   **访问控制机制** 专用于实现访问控制服务；
    *   **公证机制** 是抗抵赖服务的关键支撑。

### 4\. **TLS握手协议的认证模式的主要目标及过程**

#### **主要目标**：

*   实现通信双方的**身份认证**；
*   **协商加密算法**；
*   **交换并生成会话密钥**用于后续加密通信。

#### **认证模式**：

1.  **匿名认证模式**：无身份认证；
2.  **Client对Server认证模式**：仅服务器被认证；
3.  **相互认证模式**：双方互相认证。

#### **相互认证过程简述**（PPT图3-32）：

1.  Client → Server：ClientHello（含密码套件、随机数 NC）；
2.  Server → Client：ServerHello + Server证书 + Server密钥交换 + ServerHelloDone；
3.  Client → Server：Client证书 + Client密钥交换 + Finished；
4.  Server → Client：Finished；
5.  双方开始**加密应用数据传输**。

### 5\. **信息系统安全体系结构的主要内容**

信息系统安全体系由三部分构成：

1.  **技术体系**
    *   包括：物理安全技术、系统安全技术、OSI安全技术、相关技术管理。
2.  **组织机构体系**
    *   包括：**机构**（决策/管理/执行三层）、**岗位**、**人事**（培训、考核、监管）。
3.  **管理体系**
    *   包括：**法律**（国家法规）、**制度**（内部规章）、**培训**（安全知识教育）。

> 三者共同构成“**三位一体**”的安全保障框架，覆盖技术、组织与管理维度。
