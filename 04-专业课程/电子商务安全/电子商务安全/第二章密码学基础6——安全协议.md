# 第二章密码学基础6——安全协议
根据提供的 PPT 内容，简洁回答如下：

### 1\. 什么是安全协议？安全协议有哪些特点？

*   **定义**：安全协议是两方或多方为完成某项安全任务（如身份认证、密钥协商等）所执行的一系列有序步骤。
*   **特点**：
    *   是一个**有序的过程**，步骤必须依次执行；
    *   **至少有两个参与者**；
    *   **能完成特定安全目标**（如认证、保密、不可否认等）。

* * *

### 2\. 安全协议通常由哪些部分构成？

PPT 中未明确列出“通用组成部分”，但根据安全协议实例（如 Diffie-Hellman、SET、AKA）可归纳为：

*   **参与实体**（如用户、服务器、CA 等）；
*   **先决条件**（如共享密钥、公钥基础设施等）；
*   **交互步骤**（消息传递、计算、验证等）；
*   **安全目标**（如认证、密钥协商、支付授权）。

> （注：PPT 侧重协议实例，未抽象出统一结构）

* * *

### 3\. 简述 Diffie-Hellman 密钥交换协议的主要过程

1.  $双方 A、B 公开共享参数：素数 ( q ) 和本原元 ( \alpha )；$
2.  $A 选私钥 ( X_A )，计算公钥 ( Y_A = \alpha^{X_A} \bmod q )，发送给 B；$
3.  $B 选私钥 ( X_B )，计算公钥 ( Y_B = \alpha^{X_B} \bmod q )，发送给 A；$
4.  $A 计算共享密钥：( K = Y_B^{X_A} \bmod q )；$
5.  $B 计算共享密钥：( K = Y_A^{X_B} \bmod q )；$
6.  $由于 ( K = \alpha^{X_A X_B} \bmod q )，双方获得相同会话密钥。$

### 4\. 简化的 SET 协议（Lu-Smolka 协议）如何保证安全支付？

通过以下机制保障安全：

*   $基于 PKI：使用公钥证书和数字签名确保身份真实；$
*   $消息签名：消费者 C 和商家 M 分别用私钥签名（如 ( S_C{TID} )），防止抵赖；$
*   $敏感信息加密：客户账户信息用支付网关公钥加密（如 ( {TID, C_ACCOUNT}_{K_P} )），仅网关可解密；$
*   $订单与支付分离：OI（订单信息）和 PA/PY（金额）分别加密/签名，防止篡改；$
*   $端到端认证：三方（C、M、P）通过数字签名和加密交互，确保完整性、身份认证和机密性。$

> 协议过程共 6 步，每步均含签名或加密，实现**认证 + 保密 + 不可否认**。
